# 設計方針

## 制作のきっかけ
購入した本やグッズを忘れて、同じものを買ってしまった経験から開発を始めました。
また、予約商品の確認や、推し活に使った金額の計算を別々に行うのは手間がかかるため、まとめて管理できるアプリを目指しました。

## 解決したい課題
- 購入済みの商品を忘れてしまう
- 予約した商品を把握しづらい
- 推し活に使った金額が分かりにくい
- 入力が面倒だと記録を続けられない

## 使いやすさの工夫
最低限の情報でも登録できるようにし、入力途中の内容は下書きとして保存できるようにしました。
かぶりを検知した場合も、自動で登録を中止せず、複数購入するかどうかをユーザーが選べるようにしています。
