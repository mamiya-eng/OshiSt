# DB設計
## 基本構成
ユーザーごとに、シリーズやアイテムを管理できる構成にしています。

## テーブルを分けた理由
シリーズやカテゴリなどを整理して管理し、検索や集計をしやすくするため、アイテムとは別のテーブルに分けています。

## 複数キャラクターの管理
1つの商品に複数のキャラクターが含まれる場合があるため、`item_characters`という中間テーブルを使用しています。
これにより、1つのアイテムに複数のキャラクターを登録できます。

## 支出の集計
アイテムの購入日・金額・所持数を保存し、「金額 × 所持数」で支出額を計算します。
下書き中のアイテムは、支出の集計に含めないようにしています。
